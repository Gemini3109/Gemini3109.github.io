第一章：互联网与产品经理

1、互联网改变了什么

  • 改变空间
    • 改变人与信息的空间（存储信息/媒体介质）
    • 改变信息与信息的空间（信息间的连接）
    • 改变人与物的空间（虚拟物品）
    • 改变物与物的空间（物联网）
    • 改变人与人的空间（社交）
  • 改变时间
    • 时间被缩短
    • 时间被解放
    • 时间开始值钱

2、从招聘和应聘角度看产品

举例：产品经理JD

  • a、岗位职责：

    • 负责收集，整理，分析行业市场动态及竞品信息，为公司产品设计，研发提供支持；
    • 负责产品市场调研，进行用户研究和需求分析，对产品进行持续化优化；
    • 编写需求文档，产品原型设计等；
    • 协助项目经理跟进项目进度，协调研发，设计，测试等部门，保障项目顺利进行。

  • b、岗位要求：

    • 本科及以上学历，有相关工作经验一年以上，优秀毕业生亦可考虑；
    • 了解互联网/移动互联网行业，熟悉Ios/Android移动端产品，有APP项目开发经验者优先考虑；
    • 了解产品整体实现过程，对新概念，新产品有高度的敏感性；
    • 喜欢思考，逻辑思维清晰，具有一定的调研与分析研究能力；
    • 具有良好的沟通与协调能力，良好的执行力与团队合作意识；
    • 熟练使用word/excel/ppt/visio/mindmanager/axure等工具。

总结上述需求，大致分为两大类：

【1】沟通与逻辑能力
【2】需求分析与文档撰写

不同阶段的公司注重的能力：

  1. 初创型企业：更看重学习能力/原型制作与交互设计
  2. 成长型企业：更看重学习能力/多从运营上思考，提升用户量
  3. 成熟型企业：更看重团队合作精神/需求分析与文档撰写

3、产品经理的核心技能

【1】需求收集

需求来源包括：产品规划、领导决策、市场竞品、用户反馈、运营目标、体验问题、Bug修复。

用到搜索能力，调查产品市场反馈、收集竞品迭代功能、查看行业走向
用到沟通能力，进行用户访谈，了解领导意图、满足运营需求、在伪需求中挖掘真正的需求
用到数据分析能力，查看数据走向、发现产品问题、转化产品需求
用到逻辑思考能力，拆解需求、判断需求合理性

【2】需求分析

用到逻辑思考能力，随需求归类汇总，排列优先级，梳理满足需求的业务流程
用到工具使用能力，整理思路(XMind)/绘制原型(Axure)/录入Excel表格
需要用到设计能力，定义交互方式

【3】需求评审

产出功能列表和交互线框图
沟通能力：召开评审会议，表达产品设计思路，确认理解一致，提炼正确意见
逻辑思考能力：进行意见汇总，整理，将意见重新转化成功能列表

【4】需求跟进

已确认的需求，进入到设计/研发阶段
沟通能力：定期和各角色负责人确认进度，了解问题，查漏补缺
设计能力：评估设计稿优劣并给出修改意见
管理能力：进行项目排期，习惯任务分工，标记任务进展
工具使用能力：项目管理工具使用
逻辑思考能力：档进度超前，或进度延期时，需要结合当前研发现状，思考解决方案

【5】需求上线

开发完成的需求，最终上线
数据分析能力：收集上线后的数据表现，和预期数据对比分析，验证假设，归因问题，寻找解决方案
逻辑思考能力：一方面要确保上线流程，跟进上线操作；另一方面，针对上线表现进行总结，复盘，从拆解出新需求，归类录入需求池

PS.塑造逻辑思维习惯：脑图/UML/金字塔原理

接到需求时，习惯先用脑图将关键词拆解，层层下钻出每个分支，接下来，将分支节点进行串联，限定边界条件，通过UML图绘制，还原出每个用户场景，最后，利用金字塔原理中讲述的方法论，如：界定/归类/分层/梯队化/交叉串联/变量赋值等，把整理出来的内容表达出来。

4、如何提高逻辑思考能力

【1】学习抽象化能力
抽象化，就是认知事物本质和规律的能力

【2】学习界定能力
对概念清晰做界定，减少概念的含糊程度，让每个人理解不出现偏差，是界定的目的

【3】学习拆解能力
事物属性是多层次的，借助基本的思维模式，可以尝试对其钻研，挖取根本

【4】学习重组能力
归纳出粒度最细的节点，对其进行整合，重组新的模块

【5】学习流程化能力

流程化两个关键词，一个是“遍历”；一个是“闭环”

“遍历“–考虑到所有正常+异常的分支，描绘出所有状态下的跳转路径，争取做到场景无遗漏。
“闭环“–引入反馈机制的逻辑运行流程
【多做产品分析练习和数据分析练习】

【6】产品分析练习

寻找市面上优秀的成型产品，对其进行拆解，工具上可借助脑图梳理脉络，这样做一方面锻炼逻辑思维，另一方面了解竞品，了解行业市场走向。

作者个人习惯将一个产品拆解成：产品定位、目标用户、解决需求、产品架构、信息架构、数据表现、运营体系、商业模式、亮点、竞品、未来发展方向等几个纬度，每个维度还可以再进行拆解

举例：你在负责一款社区产品，最近在考虑提升女性用户群体验，就可以先总结几个关键词，如“女生社区”，以此去市场搜索，根据市场评星、社区活跃度、产品体验良好程度、筛选相对优秀的产品，如“她社区”，进行体验

体验步骤：
1）把自己当成用户，注册用户，在产品内浏览内容，参与互动，发表言论，积极分享，让那个自己沉浸在产品中，以此来体会产品的功能点，结构逻辑和亮点
2）把自己当成“她社区”的产品经理，思考产品为什么能满足用户需求，都通过哪些运营手段促进用户活跃，通过哪些方式为公司产生收入
3）把自己当成记者，调查产品发展历程，学习CEO对产品理念的解读，以及对行业发展的看法。

【7】数据分析练习

数据分析的练习，可针对自家产品数据进行趋势判断，挖掘变化原因，产出方案报告

举例：以作者在网易新闻的日常为例，经常需要观察的有三类数据：
1）核心产品数据
新增用户，日活跃用户，总PV，重点频道PV，次日留存率，日均使用时长等。
核心产品数据的观察方式主要是时间对比，昨日和前2日对比，和上周同期对比。
对比后输出上升/下降的数值，以此数值和系统报警阀值对比，在阀值内属于正常，阀值外，则要影响核心数据的部门讨论。

2）关键路径数据
将产品的几个主要页面抽出来，以关键路径形式组成观察漏斗，观察用户从启动app，到切换频道，到打开页面详情，到点赞、分享、撰写评论，都分别流失了多少比例。
同样将每层的流失率，和正常流失率做对比，过多和过少，都需要分析背后的原因，是页面设计问题？还是文章写的不够吸引人？还是分享按钮不够明显等。

3）线上实验数据
有时产品经理会针对特定目标，制定不同的产品方案，分发到不同渠道进行a/b测试，验证用户喜好。此时就需要观察不同产品方案的用户相应数据，这样的数据需要持续记录一段时间，每日观察，记录增长/下降变化，以此来知道产品全量上线的方案选择。

还有一种简单的办法，就是根据市面上已有产品的商业目标进行数据拆解。

例如：搜索引擎类产品的目标是提高搜索收入，那么搜索收入的计算方法可以拆解为：
搜索收入=用户搜索量 x 广告展现率 x 广告点击率 x 平均广告点击价格
接下来分别分析每个因子的数据变化影响因素，再进一步拆解：
广告展现率=每搜索出现次数/搜索总次数
而每搜索出现次数又是由关键字匹配和竞价策略决定的，这样层层下钻，既能训练逻辑推理能力，又能探求产品的商业化本质。
